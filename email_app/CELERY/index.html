    Celery Tasks Usage Guide¶

    This library provides optional Celery tasks for asynchronous email sending. The tasks are completely optional - you can use the synchronous EmailService directly if you prefer.

    Installation¶

    Option 1: Install with Celery support¶

    pip install tinkun_email_app[celery]

    This will install:

    • celery>=5.6.2
    • redis>=7.1.1

    Option 2: Manual installation¶

    pip install tinkun_email_app
pip install celery redis

    Setup¶

    1. Configure Celery in your Django project¶

    # your_project/celery.py
import os
from celery import Celery

os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'your_project.settings')

app = Celery('your_project')
app.config_from_object('django.conf:settings', namespace='CELERY')
app.autodiscover_tasks()

    2. Add Celery settings to your Django settings¶

    # settings.py

# Celery Configuration
REDIS_URL = env.str("REDIS_URL", "")
CELERY_BROKER_URL = REDIS_URL
CELERY_RESULT_BACKEND = "django-db"
CELERY_RESULT_EXTENDED = True

# Optional: Configure retry settings for email tasks
EMAIL_TASK_MAX_RETRIES = 3
EMAIL_TASK_RETRY_DELAY = 60  # seconds

    3. Initialize Celery in your app¶

    # your_project/__init__.py
from .celery import app as celery_app

__all__ = ('celery_app',)

    Usage Examples¶

    Example 1: Send Single Email Asynchronously¶

    from tinkun_email.services import EmailService
from tinkun_email.tasks.tasks import send_email_async

# In your view or service
def send_welcome_email(user_email, user_name):
    email_service = EmailService()

    # Check cooldown
    if not email_service.can_send_email_again(
        email_type="welcome",
        email_to=user_email
    ):
        return {"error": "Email sent too recently"}

    # Create audit record
    email_audit = email_service.create_email_audit(
        email_type="welcome",
        source="user_registration",
        email_to=user_email,
    )

    # Queue the email asynchronously
    result = send_email_async.delay(
        filename="welcome_email",
        email_to=user_email,
        email_type="welcome",
        subject="Welcome to Our Platform!",
        context={"user_name": user_name},
        email_audit_id=email_audit.id,
        use_txt=False,
    )

    return {
        "email_audit_id": email_audit.id,
        "task_id": result.id,
        "status": "queued"
    }

    Example 2: Send Batch Emails Asynchronously¶

    from tinkun_email.services import EmailService
from tinkun_email.tasks.tasks import send_email_batch_async

def send_newsletter_batch(user_emails, newsletter_content):
    email_service = EmailService()

    # Create batch audit records
    batch_number = email_service.create_batch_email_audits(
        email_list=user_emails,
        email_type="newsletter",
        source="marketing_campaign",
    )

    # Prepare user-specific context
    user_contexts = [
        {"unsubscribe_token": f"token_{i}"}
        for i in range(len(user_emails))
    ]

    # Queue the batch email
    result = send_email_batch_async.delay(
        filename="newsletter_template",
        email_list=user_emails,
        email_type="newsletter",
        subject="Monthly Newsletter",
        user_specific_context_list=user_contexts,
        batch_number=batch_number,
        global_context=newsletter_content,
        use_txt=True,
    )

    return {
        "batch_number": batch_number,
        "task_id": result.id,
        "email_count": len(user_emails),
        "status": "queued"
    }

    Example 3: Check Task Status¶

    from celery.result import AsyncResult

def check_email_status(task_id):
    result = AsyncResult(task_id)

    if result.ready():
        if result.successful():
            data = result.get()
            return {
                "status": "completed",
                "success": data["success"],
                "status_code": data["status_code"],
                "email_to": data.get("email_to"),
            }
        else:
            return {
                "status": "failed",
                "error": str(result.info),
            }
    else:
        return {
            "status": "pending",
            "state": result.state,
        }

    Running Celery Worker¶

    Start a Celery worker to process the tasks:

    # Development
celery -A your_project worker --loglevel=info

# Production with multiple workers
celery -A your_project worker --loglevel=info --concurrency=4

# With beat scheduler (for periodic tasks)
celery -A your_project worker --beat --loglevel=info

    Monitoring¶

    You can use Flower for monitoring Celery tasks:

    pip install flower
celery -A your_project flower

    Then visit http://localhost:5555 to see the Flower dashboard.

    Retry Behavior¶

    The tasks automatically retry on failure:

    • Max retries: 3 (configurable via EMAIL_TASK_MAX_RETRIES)
    • Retry delay: 60 seconds (configurable via EMAIL_TASK_RETRY_DELAY)
    • Exponential backoff: Enabled
    • Auto-retry exceptions: AutoretryException from tinkun_utils

    The tasks will NOT retry when:

    • HTTP 200 (Success)
    • HTTP 429 (Rate limited - will be handled by next attempt)

    The tasks WILL retry when:

    • Response is None
    • HTTP status is not 200 or 429
    • Service returns has_to_retry=True

    Troubleshooting¶

    Task not executing¶

    1. Ensure Celery worker is running
    2. Check Redis connection
    3. Verify task is registered: celery -A your_project inspect registered

    Import errors¶

    Make sure Celery dependencies are installed:

    pip install tinkun_email_app[celery]

    Alternative: Define Custom Tasks¶

    If you need custom behavior, you can define your own tasks in your project:

    # your_project/tasks.py
from celery import shared_task
from tinkun_email.services import EmailService

@shared_task(max_retries=5, rate_limit='100/h')
def custom_send_email(email_to, **kwargs):
    service = EmailService()
    # Your custom logic here
    return service.send_email(email_to=email_to, **kwargs)

    This gives you full control over retry behavior, rate limiting, and task routing.
