    REST API¶

    This guide covers the RESTful API endpoints for OIDC authentication.

    Overview¶

    The application provides four REST API endpoints:

    • POST /api/oidc/authenticate/ - Initiate OIDC authentication flow
    • POST /api/oidc/callback/ - Handle OIDC callback and complete authentication
    • POST /api/oidc/logout/ - Logout and revoke tokens
    • GET /api/oidc/user/ - Get authenticated user information (protected)

    URL Configuration¶

    Enable API Endpoints¶

    # urls.py
from django.urls import path, include

urlpatterns = [
    path('api/oidc/', include('tinkun_oidc.urls')),
]

    This mounts all OIDC endpoints under /api/oidc/.

    Custom URL Prefix¶

    urlpatterns = [
    path('auth/', include('tinkun_oidc.urls')),  # Mounted at /auth/
]

    Authentication Endpoints¶

    POST /api/oidc/authenticate/¶

    Initiates the OIDC authentication flow by generating an authorization URL.

    Request:

    GET /api/oidc/authenticate/?login_hint=user@example.com

    Query Parameters:

    Parameter Type Required Description
    login_hint string No Pre-fill user's email in Okta login form

    Response:

    {
    "redirect_url": "https://your-domain.okta.com/oauth2/v1/authorize?client_id=..."
}

    Cookies Set:

    • TKNS - State parameter for CSRF protection
    • TKNN - Nonce for replay protection (if enabled)
    • TKNV - PKCE code verifier (if enabled)

    Example:

    // Frontend code
const response = await fetch('/api/oidc/authenticate/?login_hint=user@example.com', {
    method: 'GET',
    credentials: 'include'  // Important: Include cookies
});

const data = await response.json();

// Redirect user to Okta
window.location.href = data.redirect_url;

    POST /api/oidc/callback/¶

    Handles the OIDC callback after user authentication at Okta.

    Request:

    POST /api/oidc/callback/
Content-Type: application/json

    {
    "code": "authorization_code_from_okta",
    "state": "state_from_redirect_url"
}

    Request Body:

    Field Type Required Description
    code string Yes Authorization code from Okta redirect
    state string Yes State parameter from redirect URL

    Response:

    {
    "id": 1,
    "username": "user@example.com",
    "email": "user@example.com"
}

    Cookies Set:

    • TKNA - Access token
    • TKNI - ID token
    • TKNR - Refresh token

    Cookies Deleted:

    • TKNS - State (no longer needed)
    • TKNN - Nonce (no longer needed)
    • TKNV - Code verifier (no longer needed)

    Example:

    // Frontend code - Handle Okta redirect
const urlParams = new URLSearchParams(window.location.search);
const code = urlParams.get('code');
const state = urlParams.get('state');

const response = await fetch('/api/oidc/callback/', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
    },
    credentials: 'include',  // Important: Include cookies
    body: JSON.stringify({ code, state })
});

const userData = await response.json();
console.log('Authenticated user:', userData);

// Redirect to main app
window.location.href = '/dashboard';

    Error Responses:

    // Invalid or expired code
{
    "error": "Invalid authorization code"
}

// State mismatch (CSRF attempt)
{
    "error": "Invalid state parameter"
}

// Token verification failed
{
    "error": "Token verification failed"
}

    POST /api/oidc/logout/¶

    Logs out the user by revoking tokens and clearing cookies.

    Request:

    POST /api/oidc/logout/

    Response:

    {
    "logout_url": "https://your-domain.okta.com/oauth2/v1/logout?id_token_hint=..."
}

    Cookies Deleted:

    • TKNA - Access token
    • TKNI - ID token
    • TKNR - Refresh token

    Example:

    // Frontend code
const response = await fetch('/api/oidc/logout/', {
    method: 'POST',
    credentials: 'include'
});

const data = await response.json();

// Redirect to Okta logout to end session
window.location.href = data.logout_url;

    Note: Always redirect to the logout_url to ensure the Okta session is terminated.

    Protected Endpoints¶

    GET /api/oidc/user/¶

    Get authenticated user information. Requires valid authentication cookies.

    Authentication: Required (uses DRF OIDCAuthentication)

    Request:

    GET /api/oidc/user/

    Response:

    {
    "id": 1,
    "username": "user@example.com",
    "email": "user@example.com"
}

    Example:

    // Frontend code
const response = await fetch('/api/oidc/user/', {
    method: 'GET',
    credentials: 'include'  // Cookies automatically sent
});

if (response.ok) {
    const user = await response.json();
    console.log('Current user:', user);
} else if (response.status === 401) {
    console.log('Not authenticated');
    // Redirect to login
}

    Error Responses:

    // Not authenticated
{
    "detail": "Authentication credentials were not provided."
}

// Invalid or expired token
{
    "detail": "Invalid authentication token."
}

    Automatic Token Refresh¶

    The API automatically refreshes expired access tokens using the refresh token:

    1. Request received with expired access token
    2. OIDCAuthentication detects expiration
    3. Automatically refreshes using refresh token cookie
    4. New tokens stored in temporary cookies (TKNTA, TKNTI, TKNTR)
    5. ReplaceRefreshedTokensMiddleware replaces old cookies with new tokens
    6. Response sent with updated cookies

    This happens transparently - no client-side code needed!

    Custom Serializers¶

    Output Serializers¶

    Customize the response format:

    # myapp/serializers.py
from rest_framework import serializers


class CustomUserSerializer(serializers.Serializer):
    id = serializers.IntegerField()
    username = serializers.CharField()
    email = serializers.EmailField()
    first_name = serializers.CharField()
    last_name = serializers.CharField()
    full_name = serializers.SerializerMethodField()

    def get_full_name(self, obj):
        return f"{obj.first_name} {obj.last_name}".strip()

    Configuration:

    # settings.py
TINKUN_OIDC = {
    "AUTHENTICATION_OUTPUT_SERIALIZER": "myapp.serializers.CustomUserSerializer",
    "GET_USER_OUTPUT_SERIALIZER": "myapp.serializers.CustomUserSerializer",
}

    Note: Both callback and user endpoints can use different serializers if needed.

    DRF Authentication¶

    Enable for All Views¶

    Add to DRF settings:

    # settings.py
REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'tinkun_oidc.drf.authentication.OIDCAuthentication',
    ],
}

    Use in Custom Views¶

    # myapp/views.py
from rest_framework.views import APIView
from rest_framework.permissions import IsAuthenticated
from rest_framework.response import Response

from tinkun_oidc.drf.authentication import OIDCAuthentication


class MyProtectedView(APIView):
    authentication_classes = [OIDCAuthentication]
    permission_classes = [IsAuthenticated]

    def get(self, request):
        user = request.user
        return Response({
            'message': f'Hello {user.email}!',
            'user_id': user.id
        })

    Access User in View¶

    def get(self, request):
    # User is automatically set by OIDCAuthentication
    user = request.user

    # Check if authenticated
    if user.is_authenticated:
        return Response({'email': user.email})
    else:
        return Response({'error': 'Not authenticated'}, status=401)

    Middleware¶

    Token Refresh Middleware¶

    The middleware replaces temporary tokens with permanent cookies after refresh:

    # settings.py
MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.common.CommonMiddleware',
    'django.middleware.csrf.CsrfViewMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
    'django.middleware.clickjacking.XFrameOptionsMiddleware',

    # Add OIDC middleware
    'tinkun_oidc.drf.middleware.ReplaceRefreshedTokensMiddleware',
]

    Important: Place after AuthenticationMiddleware.

    Frontend Integration¶

    Complete Authentication Flow¶

    // 1. Initiate authentication
async function login() {
    const response = await fetch('/api/oidc/authenticate/', {
        method: 'GET',
        credentials: 'include'
    });

    const data = await response.json();
    window.location.href = data.redirect_url;
}

// 2. Handle callback (on redirect page)
async function handleCallback() {
    const urlParams = new URLSearchParams(window.location.search);
    const code = urlParams.get('code');
    const state = urlParams.get('state');

    const response = await fetch('/api/oidc/callback/', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        credentials: 'include',
        body: JSON.stringify({ code, state })
    });

    if (response.ok) {
        const user = await response.json();
        // Redirect to app
        window.location.href = '/dashboard';
    } else {
        console.error('Authentication failed');
    }
}

// 3. Get current user
async function getCurrentUser() {
    const response = await fetch('/api/oidc/user/', {
        method: 'GET',
        credentials: 'include'
    });

    if (response.ok) {
        return await response.json();
    }
    return null;
}

// 4. Logout
async function logout() {
    const response = await fetch('/api/oidc/logout/', {
        method: 'POST',
        credentials: 'include'
    });

    const data = await response.json();
    window.location.href = data.logout_url;
}

// 5. Make authenticated API calls
async function fetchProtectedData() {
    const response = await fetch('/api/my-endpoint/', {
        method: 'GET',
        credentials: 'include'  // Cookies sent automatically
    });

    return await response.json();
}

    Error Handling¶

    Standard Error Responses¶

    All endpoints follow DRF error response format:

    {
    "detail": "Error message here"
}

    Or with field-specific errors:

    {
    "code": ["This field is required."],
    "state": ["This field is required."]
}

    HTTP Status Codes¶

    Status Meaning
    200 Success
    400 Bad request (validation error)
    401 Unauthorized (not authenticated)
    403 Forbidden (authenticated but no permission)
    500 Server error
