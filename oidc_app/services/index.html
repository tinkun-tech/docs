    Service Layer Architecture¶

    This guide covers the service-oriented architecture and how to customize the business logic layer.

    Overview¶

    The application uses a clean service-oriented architecture with dedicated service classes for different responsibilities:

    • OIDCService - Main facade coordinating all OIDC operations
    • OIDCAuthenticationService - Handles authorization URL generation and callback processing
    • OIDCTokenService - Manages token exchange, refresh, and revocation
    • OIDCUserService - User creation and management
    • OIDCJWTVerifierService - JWT token verification

    Service Architecture¶

    Separation of Concerns¶

    Each service has a single, well-defined responsibility:

    ┌─────────────────┐
│   OIDCService   │  ← Main facade
└────────┬────────┘
         │
         ├──→ OIDCAuthenticationService  (Authorization flow)
         ├──→ OIDCTokenService           (Token operations)
         ├──→ OIDCUserService            (User management)
         └──→ OIDCJWTVerifierService     (JWT verification)

    Default Services¶

    All services work out of the box without customization:

    # settings.py
TINKUN_OIDC = {
    "OKTA_DOMAIN": "your-domain.okta.com",
    "OIDC_RP_CLIENT_ID": "your-client-id",
    "OIDC_RP_CLIENT_SECRET": "your-client-secret",
    # Services use default implementations
}

    OIDCService (Main Facade)¶

    The main service that coordinates all OIDC operations.

    Default Implementation¶

    from tinkun_oidc.services.oidc_service import OIDCService

# Usage in views
oidc_service = OIDCService()

# Basic usage
redirect_url, state, nonce, code_verifier = oidc_service.get_authorization_url()

# With optional parameters
redirect_url, state, nonce, code_verifier = oidc_service.get_authorization_url(
    login_hint="user@example.com",
    prompt="login",
    langCode="es",  # Extra provider-specific param
)

    Custom Implementation¶

    # myapp/services.py
from tinkun_oidc.services.oidc_service import OIDCService as BaseOIDCService


class CustomOIDCService(BaseOIDCService):

    def get_authorization_url(
        self, login_hint: str = None, prompt: str = None, **extra_params
    ):
        """Add custom logging or analytics"""
        # Log authentication attempt
        logger.info(f"Authentication initiated for: {login_hint}")

        # Call parent implementation
        result = super().get_authorization_url(login_hint, prompt, **extra_params)

        # Track analytics
        self.track_auth_start(login_hint)

        return result

    def track_auth_start(self, login_hint: str):
        """Custom analytics tracking"""
        # Your analytics code
        pass

    Configuration:

    TINKUN_OIDC = {
    "OIDC_SERVICE": "myapp.services.CustomOIDCService",
}

    Available Methods¶

    Method Description
    get_authorization_url(login_hint=None, prompt=None, **extra_params) Generate authorization URL with state, nonce, and PKCE
    authenticate_callback(code, state, code_verifier, nonce) Process callback and return user
    logout(refresh_token) Revoke tokens and generate logout URL

    OIDCAuthenticationService¶

    Handles authorization URL generation and callback processing.

    Custom Implementation¶

    # myapp/services.py
from tinkun_oidc.services.auth_service import OIDCAuthenticationService as BaseAuthService


class CustomAuthenticationService(BaseAuthService):

    def generate_authorization_url(
        self, state: str, nonce: str = None, code_challenge: str = None, login_hint: str = None
    ):
        """Add custom query parameters to authorization URL"""
        url = super().generate_authorization_url(state, nonce, code_challenge, login_hint)

        # Add custom parameters
        url += "&prompt=login"  # Force re-authentication
        url += "&max_age=0"     # Require fresh authentication

        return url

    def process_callback(self, code: str, state: str, code_verifier: str = None, nonce: str = None):
        """Add custom callback processing"""
        # Custom pre-processing
        self.log_callback_received(code)

        # Call parent implementation
        tokens = super().process_callback(code, state, code_verifier, nonce)

        # Custom post-processing
        self.track_successful_login(tokens)

        return tokens

    Configuration:

    TINKUN_OIDC = {
    "OIDC_AUTHENTICATION_SERVICE": "myapp.services.CustomAuthenticationService",
}

    Available Methods¶

    Method Description
    generate_state() Generate cryptographically random state
    generate_nonce() Generate cryptographically random nonce
    generate_pkce_parameters() Generate PKCE code verifier and challenge
    generate_authorization_url(...) Build complete authorization URL
    process_callback(...) Exchange code for tokens and verify

    OIDCTokenService¶

    Manages token operations including exchange, refresh, and revocation.

    Custom Implementation¶

    # myapp/services.py
from tinkun_oidc.services.token_service import OIDCTokenService as BaseTokenService


class CustomTokenService(BaseTokenService):

    def exchange_code_for_tokens(self, code: str, code_verifier: str = None):
        """Add token caching or logging"""
        tokens = super().exchange_code_for_tokens(code, code_verifier)

        # Cache tokens in Redis for faster lookup
        self.cache_tokens(tokens)

        # Log successful token exchange
        logger.info("Tokens exchanged successfully")

        return tokens

    def refresh_tokens(self, refresh_token: str):
        """Add custom refresh logic"""
        try:
            tokens = super().refresh_tokens(refresh_token)

            # Update cache with new tokens
            self.update_cached_tokens(tokens)

            return tokens
        except Exception as e:
            # Custom error handling
            logger.error(f"Token refresh failed: {e}")
            self.notify_admin_of_refresh_failure(e)
            raise

    def revoke_refresh_token(self, refresh_token: str):
        """Add cleanup logic"""
        # Clear from cache before revoking
        self.clear_cached_tokens(refresh_token)

        # Revoke with provider
        super().revoke_refresh_token(refresh_token)

        # Additional cleanup
        self.cleanup_user_sessions(refresh_token)

    Configuration:

    TINKUN_OIDC = {
    "OIDC_TOKEN_SERVICE": "myapp.services.CustomTokenService",
}

    Available Methods¶

    Method Description
    exchange_code_for_tokens(code, code_verifier=None) Exchange authorization code for tokens
    refresh_tokens(refresh_token) Get new access and ID tokens
    revoke_refresh_token(refresh_token) Revoke refresh token with provider

    OIDCUserService¶

    Handles user creation and updates based on token claims.

    Custom Implementation¶

    # myapp/services.py
from django.contrib.auth import get_user_model

from tinkun_oidc.services.user_service import OIDCUserService as BaseUserService

User = get_user_model()


class CustomUserService(BaseUserService):

    def get_or_create_user(self, claims: dict):
        """Custom user creation with profile setup"""
        user, created = super().get_or_create_user(claims)

        if created:
            # Set up user profile
            self.create_user_profile(user, claims)

            # Send welcome email
            self.send_welcome_email(user)

            # Assign default permissions
            self.assign_default_permissions(user)

            # Track new user signup
            self.track_new_user(user)

        else:
            # Update existing user
            self.update_user_info(user, claims)

        return user

    def create_user(self, claims: dict):
        """Custom user creation logic"""
        user = User.objects.create_user(
            username=claims.get("sub"),
            email=claims.get("email"),
            first_name=claims.get("given_name", ""),
            last_name=claims.get("family_name", ""),
        )

        # Set additional fields from claims
        user.profile.department = claims.get("department", "")
        user.profile.job_title = claims.get("job_title", "")
        user.profile.phone = claims.get("phone", "")
        user.save()

        return user

    def update_user_info(self, user, claims: dict):
        """Update user information on login"""
        user.email = claims.get("email")
        user.first_name = claims.get("given_name", "")
        user.last_name = claims.get("family_name", "")
        user.profile.department = claims.get("department", "")
        user.save()

    Configuration:

    TINKUN_OIDC = {
    "OIDC_USER_SERVICE": "myapp.services.CustomUserService",
    "OIDC_CREATE_USER": True,
}

    Available Methods¶

    Method Description
    get_or_create_user(claims) Get existing or create new user from token claims
    create_user(claims) Create new user from claims
    get_username_from_claims(claims) Extract username (default: sub claim)

    OIDCJWTVerifierService¶

    Verifies JWT tokens using Okta's JWT verifier.

    Custom Implementation¶

    # myapp/services.py
from tinkun_oidc.services.jwt_service import OIDCJWTVerifierService as BaseJWTService


class CustomJWTVerifierService(BaseJWTService):

    def verify_id_token(self, id_token: str, nonce: str = None):
        """Add custom ID token validation"""
        claims = super().verify_id_token(id_token, nonce)

        # Custom validation
        if not claims.get("email_verified"):
            raise ValueError("Email not verified")

        # Check allowed domains
        email = claims.get("email", "")
        if not email.endswith("@yourcompany.com"):
            raise ValueError("Only company emails allowed")

        return claims

    def verify_access_token(self, access_token: str):
        """Add custom access token validation"""
        claims = super().verify_access_token(access_token)

        # Validate required scopes
        scopes = claims.get("scp", [])
        if "api:access" not in scopes:
            raise ValueError("Missing required scope: api:access")

        # Check token age
        issued_at = claims.get("iat")
        if self.is_token_too_old(issued_at):
            raise ValueError("Token is too old, please re-authenticate")

        return claims

    def is_token_too_old(self, issued_at: int) -> bool:
        """Check if token was issued more than 24 hours ago"""
        from datetime import datetime, timedelta
        import time

        token_age = datetime.fromtimestamp(time.time()) - datetime.fromtimestamp(issued_at)
        return token_age > timedelta(hours=24)

    Configuration:

    TINKUN_OIDC = {
    "OIDC_JWT_VERIFIER_SERVICE": "myapp.services.CustomJWTVerifierService",
}

    Available Methods¶

    Method Description
    verify_id_token(id_token, nonce=None) Verify ID token and return claims
    verify_access_token(access_token) Verify access token and return claims

    Token Claims Validation¶

    Configure required claims:

    TINKUN_OIDC = {
    "OIDC_ID_TOKEN_CLAIMS": ("iss", "exp", "iat", "sub", "email", "email_verified"),
    "OIDC_ACCESS_TOKEN_CLAIMS": ("iss", "aud", "exp", "uid", "scp"),
}

    Service Interfaces¶

    All services implement abstract interfaces for type safety and consistency:

    from tinkun_oidc.interfaces.oidc_service_interface import AbstractOIDCService
from tinkun_oidc.interfaces.auth_service_interface import AbstractOIDCAuthenticationService
from tinkun_oidc.interfaces.token_service_interface import AbstractOIDCTokenService
from tinkun_oidc.interfaces.user_service_interface import AbstractOIDCUserService
from tinkun_oidc.interfaces.jwt_service_interface import AbstractOIDCJWTVerifierService

    When creating custom services, inherit from the appropriate interface to ensure compatibility.
