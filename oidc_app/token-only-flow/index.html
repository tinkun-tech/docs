    Token-Only Flow¶

    Get Okta tokens without creating users in Django.

    Overview¶

    • ✅ Tokens stored in HTTP-only cookies + returned in response
    • ✅ Optional automatic token refresh
    • ❌ No Django users created

    Setup¶

    URLs¶

    # urls.py
from tinkun_oidc.views import OIDCAuthenticationRequestView, OIDCTokenOnlyCallbackView

urlpatterns = [
    path('api/oidc/authenticate/', OIDCAuthenticationRequestView.as_view()),
    path('api/oidc/callback/', OIDCTokenOnlyCallbackView.as_view()),
]

    Settings¶

    # settings.py
INSTALLED_APPS = ['tinkun_oidc', 'rest_framework']

TINKUN_OIDC = {
    "OKTA_DOMAIN": "your-domain.okta.com",
    "OIDC_RP_CLIENT_ID": "your-client-id",
    "OIDC_RP_CLIENT_SECRET": "your-client-secret",
}

    Optional: Auto Token Refresh¶

    MIDDLEWARE = [
    # ...
    'tinkun_oidc.drf.middleware.ReplaceRefreshedTokensMiddleware',
]

REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'tinkun_oidc.drf.authentication.OIDCAuthentication',
    ],
}

    Frontend Integration¶

    JavaScript¶

    // 1. Login
async function login() {
    const res = await fetch('/api/oidc/authenticate/', { credentials: 'include' });
    const { redirect_url } = await res.json();
    window.location.href = redirect_url;
}

// 2. Handle callback (Okta redirects here with code and state)
async function handleCallback() {
    // Code and state are in URL, cookies sent automatically
    const res = await fetch(window.location.href.replace(window.location.origin, '/api/oidc'), {
        credentials: 'include'
    });

    const tokens = await res.json();
    // Tokens in cookies automatically + available in response
    return tokens;
}

// 3. Use cookies (automatic)
async function callAPI(endpoint) {
    return fetch(endpoint, { credentials: 'include' });
}

// 4. Or use tokens manually (for external APIs)
async function callExternalAPI(endpoint, accessToken) {
    return fetch(endpoint, {
        headers: { 'Authorization': `Bearer ${accessToken}` }
    });
}

// 5. Logout
async function logout() {
    const res = await fetch('/api/oidc/logout/', {
        method: 'POST',
        credentials: 'include'
    });
    const { logout_url } = await res.json();
    window.location.href = logout_url;
}

    React¶

    import { useEffect, useState } from 'react';

function App() {
    const [isAuth, setIsAuth] = useState(false);

    useEffect(() => {
        // Handle callback
        const params = new URLSearchParams(window.location.search);
        if (params.get('code')) {
            handleCallback().then(() => {
                setIsAuth(true);
                window.history.replaceState({}, '', '/');
            });
        }
    }, []);

    return isAuth ? <Dashboard /> : <button onClick={login}>Login</button>;
}

    Response Format¶

    Success¶

    Response body: 

    {
    "access_token": "...",
    "id_token": "...",
    "refresh_token": "...",
    "token_type": "Bearer"
}

    Cookies: TKNA, TKNI, TKNR (HTTP-only, secure)

    Error¶

    {
    "state": "Invalid state parameter"
}

    vs Standard Flow¶

    Feature Standard Token-Only
    Django user ✅ Created ❌ Not created
    Cookies ✅ ✅
    Response body ❌ ✅ Tokens included
    Auto refresh ✅ ✅ Optional

    When to Use¶

    ✅ Use when: - No user database needed - Need tokens for external APIs - Want cookie security + token flexibility

    ❌ Don't use when: - Need Django User model - Need Django permissions - Want full Django auth integration
