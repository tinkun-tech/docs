    Configuration Guide¶

    This guide covers all available configuration options for the Django OIDC Authentication package. All settings are configured through the TINKUN_OIDC dictionary in your Django settings.

    Quick Reference¶

    TINKUN_OIDC = {
    # OIDC Provider Settings (Required)
    "OKTA_DOMAIN": None,
    "OIDC_RP_CLIENT_ID": None,
    "OIDC_RP_CLIENT_SECRET": None,

    # OIDC Provider Settings (Optional)
    "OIDC_RP_SCOPES": "openid email offline_access",
    "OIDC_AUTHENTICATION_CALLBACK_URL": "http://localhost:3000/authorization-code/callback",

    # OIDC Provider Endpoints (Auto-generated)
    "OIDC_OP_AUTHORIZATION_ENDPOINT": None,
    "OIDC_OP_TOKEN_ENDPOINT": None,
    "OIDC_OP_TOKEN_REVOKE_ENDPOINT": None,
    "OIDC_OP_LOGOUT_ENDPOINT": None,
    "OIDC_TOKEN_USE_BASIC_AUTH": False,

    # Security Settings
    "OIDC_USE_NONCE": True,
    "OIDC_USE_PKCE": True,
    "OIDC_STATE_SIZE": 32,
    "OIDC_NONCE_SIZE": 32,
    "OIDC_CODE_CHALLENGE_METHOD": "S256",

    # User Management
    "OIDC_CREATE_USER": True,

    # JWT Verification
    "OIDC_ISSUER": None,  # Defaults to https://{OKTA_DOMAIN}
    "OIDC_AUDIENCE": None,  # Defaults to https://{OKTA_DOMAIN}
    "OIDC_ID_TOKEN_CLAIMS": ("iss", "exp", "iat", "sub", "email"),
    "OIDC_ACCESS_TOKEN_CLAIMS": ("iss", "aud", "exp", "uid"),

    # Cookie Names
    "OIDC_STATE_COOKIE_NAME": "TKNS",
    "OIDC_NONCE_COOKIE_NAME": "TKNN",
    "OIDC_ACCESS_TOKEN_COOKIE_NAME": "TKNA",
    "OIDC_ID_TOKEN_COOKIE_NAME": "TKNI",
    "OIDC_REFRESH_TOKEN_COOKIE_NAME": "TKNR",
    "OIDC_CODE_VERIFIER_COOKIE_NAME": "TKNV",
    "OIDC_TEMP_ACCESS_TOKEN_COOKIE_NAME": "TKNTA",
    "OIDC_TEMP_ID_TOKEN_COOKIE_NAME": "TKNTI",
    "OIDC_TEMP_REFRESH_TOKEN_COOKIE_NAME": "TKNTR",

    # Cookie Expiration
    "OIDC_REFRESH_TOKEN_COOKIE_MAX_AGE": 345600,  # 4 days

    # Services (Customization)
    "OIDC_SERVICE": "tinkun_oidc.services.oidc_service.OIDCService",
    "OIDC_AUTHENTICATION_SERVICE": "tinkun_oidc.services.auth_service.OIDCAuthenticationService",
    "OIDC_TOKEN_SERVICE": "tinkun_oidc.services.token_service.OIDCTokenService",
    "OIDC_USER_SERVICE": "tinkun_oidc.services.user_service.OIDCUserService",
    "OIDC_JWT_VERIFIER_SERVICE": "tinkun_oidc.services.jwt_service.OIDCJWTVerifierService",

    # Serializers (Customization)
    "AUTHENTICATION_OUTPUT_SERIALIZER": "tinkun_oidc.serializers.UserSerializer",
    "GET_USER_OUTPUT_SERIALIZER": "tinkun_oidc.serializers.UserSerializer",
}

    Required Settings¶

    Setting Type Description
    OKTA_DOMAIN str Your Okta domain (e.g., dev-123456.okta.com)
    OIDC_RP_CLIENT_ID str OAuth 2.0 client ID from Okta application
    OIDC_RP_CLIENT_SECRET str OAuth 2.0 client secret from Okta application

    Example:

    TINKUN_OIDC = {
    "OKTA_DOMAIN": "dev-123456.okta.com",
    "OIDC_RP_CLIENT_ID": "0oa1234567890abcdef",
    "OIDC_RP_CLIENT_SECRET": "very-secret-string-from-okta",
}

    OIDC Provider Settings¶

    Setting Type Default Description
    OIDC_RP_SCOPES str "openid email offline_access" OAuth scopes to request
    OIDC_AUTHENTICATION_CALLBACK_URL str "http://localhost:3000/authorization-code/callback" Frontend callback URL after Okta authentication

    OAuth Scopes¶

    Common scopes:

    • openid - Required for OIDC
    • email - User's email address
    • profile - User's profile information (name, etc.)
    • offline_access - Refresh token for token renewal
    • groups - User's Okta groups

    Example:

    TINKUN_OIDC = {
    "OIDC_RP_SCOPES": "openid email profile groups offline_access",
}

    Callback URL¶

    The URL where Okta redirects after authentication:

    TINKUN_OIDC = {
    # Development
    "OIDC_AUTHENTICATION_CALLBACK_URL": "http://localhost:3000/authorization-code/callback",

    # Production
    "OIDC_AUTHENTICATION_CALLBACK_URL": "https://yourapp.com/auth/callback",
}

    Important: This URL must be configured in your Okta application settings.

    OIDC Provider Endpoints¶

    These endpoints are auto-generated from OKTA_DOMAIN. Override only for custom configurations.

    Setting Type Default
    OIDC_OP_AUTHORIZATION_ENDPOINT str https://{OKTA_DOMAIN}/oauth2/v1/authorize
    OIDC_OP_TOKEN_ENDPOINT str https://{OKTA_DOMAIN}/oauth2/v1/token
    OIDC_OP_TOKEN_REVOKE_ENDPOINT str https://{OKTA_DOMAIN}/oauth2/v1/revoke
    OIDC_OP_LOGOUT_ENDPOINT str https://{OKTA_DOMAIN}/oauth2/v1/logout
    OIDC_TOKEN_USE_BASIC_AUTH bool False

    Custom Authorization Server:

    TINKUN_OIDC = {
    "OKTA_DOMAIN": "dev-123456.okta.com",
    "OIDC_OP_AUTHORIZATION_ENDPOINT": "https://dev-123456.okta.com/oauth2/default/v1/authorize",
    "OIDC_OP_TOKEN_ENDPOINT": "https://dev-123456.okta.com/oauth2/default/v1/token",
    "OIDC_OP_TOKEN_REVOKE_ENDPOINT": "https://dev-123456.okta.com/oauth2/default/v1/revoke",
    "OIDC_OP_LOGOUT_ENDPOINT": "https://dev-123456.okta.com/oauth2/default/v1/logout",
}

    Security Settings¶

    Setting Type Default Description
    OIDC_USE_NONCE bool True Enable nonce for replay attack protection
    OIDC_USE_PKCE bool True Enable PKCE for authorization code flow security
    OIDC_STATE_SIZE int 32 Size of state parameter in bytes
    OIDC_NONCE_SIZE int 32 Size of nonce parameter in bytes
    OIDC_CODE_CHALLENGE_METHOD str "S256" PKCE code challenge method (SHA-256)

    State Parameter¶

    CSRF protection using cryptographically random state:

    TINKUN_OIDC = {
    "OIDC_STATE_SIZE": 32,  # 32 bytes = 256 bits
}

    Nonce¶

    Replay attack protection:

    TINKUN_OIDC = {
    "OIDC_USE_NONCE": True,    # Recommended
    "OIDC_NONCE_SIZE": 32,     # 32 bytes = 256 bits
}

    To disable (not recommended):

    TINKUN_OIDC = {
    "OIDC_USE_NONCE": False,
}

    PKCE¶

    Enhanced authorization code flow security:

    TINKUN_OIDC = {
    "OIDC_USE_PKCE": True,                 # Strongly recommended
    "OIDC_CODE_CHALLENGE_METHOD": "S256",  # SHA-256 (required)
}

    Note: PKCE should always be enabled per OAuth 2.0 security best practices.

    User Management¶

    Setting Type Default Description
    OIDC_CREATE_USER bool True Automatically create users on first login

    Auto-Create Users¶

    TINKUN_OIDC = {
    "OIDC_CREATE_USER": True,  # Create new users automatically
}

    Existing Users Only¶

    TINKUN_OIDC = {
    "OIDC_CREATE_USER": False,  # Only authenticate existing users
}

    JWT Verification Settings¶

    Setting Type Default Description
    OIDC_ISSUER str https://{OKTA_DOMAIN} Expected token issuer
    OIDC_AUDIENCE str https://{OKTA_DOMAIN} Expected token audience
    OIDC_ID_TOKEN_CLAIMS tuple ("iss", "exp", "iat", "sub", "email") Required ID token claims
    OIDC_ACCESS_TOKEN_CLAIMS tuple ("iss", "aud", "exp", "uid") Required access token claims

    Issuer and Audience¶

    Auto-generated from OKTA_DOMAIN. Override for custom authorization servers:

    TINKUN_OIDC = {
    "OKTA_DOMAIN": "dev-123456.okta.com",
    "OIDC_ISSUER": "https://dev-123456.okta.com/oauth2/default",
    "OIDC_AUDIENCE": "api://default",
}

    Required Token Claims¶

    ID Token:

    TINKUN_OIDC = {
    "OIDC_ID_TOKEN_CLAIMS": (
        "iss",           # Issuer
        "exp",           # Expiration
        "iat",           # Issued at
        "sub",           # Subject (user ID)
        "email",         # User email
        "email_verified" # Email verification status (optional)
    ),
}

    Access Token:

    TINKUN_OIDC = {
    "OIDC_ACCESS_TOKEN_CLAIMS": (
        "iss",  # Issuer
        "aud",  # Audience
        "exp",  # Expiration
        "uid",  # User ID
        "scp",  # Scopes (optional)
    ),
}

    Cookie Settings¶

    Cookie Names¶

    Setting Type Default Description
    OIDC_STATE_COOKIE_NAME str "TKNS" State parameter cookie
    OIDC_NONCE_COOKIE_NAME str "TKNN" Nonce cookie
    OIDC_ACCESS_TOKEN_COOKIE_NAME str "TKNA" Access token cookie
    OIDC_ID_TOKEN_COOKIE_NAME str "TKNI" ID token cookie
    OIDC_REFRESH_TOKEN_COOKIE_NAME str "TKNR" Refresh token cookie
    OIDC_CODE_VERIFIER_COOKIE_NAME str "TKNV" PKCE code verifier cookie
    OIDC_TEMP_ACCESS_TOKEN_COOKIE_NAME str "TKNTA" Temporary access token (during refresh)
    OIDC_TEMP_ID_TOKEN_COOKIE_NAME str "TKNTI" Temporary ID token (during refresh)
    OIDC_TEMP_REFRESH_TOKEN_COOKIE_NAME str "TKNTR" Temporary refresh token (during refresh)

    Custom Cookie Names:

    TINKUN_OIDC = {
    "OIDC_ACCESS_TOKEN_COOKIE_NAME": "myapp_access",
    "OIDC_ID_TOKEN_COOKIE_NAME": "myapp_id",
    "OIDC_REFRESH_TOKEN_COOKIE_NAME": "myapp_refresh",
}

    Cookie Expiration¶

    Setting Type Default Description
    OIDC_REFRESH_TOKEN_COOKIE_MAX_AGE int 345600 Refresh token cookie lifetime in seconds (4 days)

    Example:

    TINKUN_OIDC = {
    # 7 days
    "OIDC_REFRESH_TOKEN_COOKIE_MAX_AGE": 60 * 60 * 24 * 7,

    # 30 days
    "OIDC_REFRESH_TOKEN_COOKIE_MAX_AGE": 60 * 60 * 24 * 30,
}

    Note: Access and ID token cookies are session cookies (deleted when browser closes).

    Service Customization¶

    Override default services with custom implementations:

    Setting Type Default
    OIDC_SERVICE str "tinkun_oidc.services.oidc_service.OIDCService"
    OIDC_AUTHENTICATION_SERVICE str "tinkun_oidc.services.auth_service.OIDCAuthenticationService"
    OIDC_TOKEN_SERVICE str "tinkun_oidc.services.token_service.OIDCTokenService"
    OIDC_USER_SERVICE str "tinkun_oidc.services.user_service.OIDCUserService"
    OIDC_JWT_VERIFIER_SERVICE str "tinkun_oidc.services.jwt_service.OIDCJWTVerifierService"

    Example:

    TINKUN_OIDC = {
    "OIDC_USER_SERVICE": "myapp.services.CustomUserService",
    "OIDC_JWT_VERIFIER_SERVICE": "myapp.services.CustomJWTVerifier",
}

    See Services for detailed customization examples.

    Serializer Customization¶

    Customize API response formats:

    Setting Type Default
    AUTHENTICATION_OUTPUT_SERIALIZER str "tinkun_oidc.serializers.UserSerializer"
    GET_USER_OUTPUT_SERIALIZER str "tinkun_oidc.serializers.UserSerializer"

    Example:

    TINKUN_OIDC = {
    "AUTHENTICATION_OUTPUT_SERIALIZER": "myapp.serializers.AuthUserSerializer",
    "GET_USER_OUTPUT_SERIALIZER": "myapp.serializers.DetailedUserSerializer",
}

    See API for serializer examples.

    Configuration Examples¶

    Minimal Setup¶

    TINKUN_OIDC = {
    "OKTA_DOMAIN": "dev-123456.okta.com",
    "OIDC_RP_CLIENT_ID": "0oa1234567890abcdef",
    "OIDC_RP_CLIENT_SECRET": "very-secret-string",
}

    Development¶

    TINKUN_OIDC = {
    "OKTA_DOMAIN": "dev-123456.okta.com",
    "OIDC_RP_CLIENT_ID": "0oa1234567890abcdef",
    "OIDC_RP_CLIENT_SECRET": "very-secret-string",
    "OIDC_AUTHENTICATION_CALLBACK_URL": "http://localhost:3000/auth/callback",
    "OIDC_RP_SCOPES": "openid email profile offline_access",
}

    Production¶

    import environ

env = environ.Env()

TINKUN_OIDC = {
    # Load from environment variables
    "OKTA_DOMAIN": env("OKTA_DOMAIN"),
    "OIDC_RP_CLIENT_ID": env("OIDC_CLIENT_ID"),
    "OIDC_RP_CLIENT_SECRET": env("OIDC_CLIENT_SECRET"),
    "OIDC_AUTHENTICATION_CALLBACK_URL": env("OIDC_CALLBACK_URL"),

    # Security settings
    "OIDC_RP_SCOPES": "openid email profile groups offline_access",
    "OIDC_USE_NONCE": True,
    "OIDC_USE_PKCE": True,

    # User management
    "OIDC_CREATE_USER": True,

    # Custom services
    "OIDC_USER_SERVICE": "myapp.services.CustomUserService",

    # Extended token lifetime
    "OIDC_REFRESH_TOKEN_COOKIE_MAX_AGE": 60 * 60 * 24 * 7,  # 7 days
}

    Custom Authorization Server¶

    TINKUN_OIDC = {
    "OKTA_DOMAIN": "dev-123456.okta.com",
    "OIDC_RP_CLIENT_ID": "0oa1234567890abcdef",
    "OIDC_RP_CLIENT_SECRET": "very-secret-string",

    # Custom authorization server endpoints
    "OIDC_OP_AUTHORIZATION_ENDPOINT": "https://dev-123456.okta.com/oauth2/default/v1/authorize",
    "OIDC_OP_TOKEN_ENDPOINT": "https://dev-123456.okta.com/oauth2/default/v1/token",
    "OIDC_OP_TOKEN_REVOKE_ENDPOINT": "https://dev-123456.okta.com/oauth2/default/v1/revoke",
    "OIDC_OP_LOGOUT_ENDPOINT": "https://dev-123456.okta.com/oauth2/default/v1/logout",

    # Custom issuer and audience
    "OIDC_ISSUER": "https://dev-123456.okta.com/oauth2/default",
    "OIDC_AUDIENCE": "api://default",
}

    Enterprise Setup¶

    TINKUN_OIDC = {
    # Provider configuration
    "OKTA_DOMAIN": env("OKTA_DOMAIN"),
    "OIDC_RP_CLIENT_ID": env("OIDC_CLIENT_ID"),
    "OIDC_RP_CLIENT_SECRET": env("OIDC_CLIENT_SECRET"),
    "OIDC_AUTHENTICATION_CALLBACK_URL": env("OIDC_CALLBACK_URL"),
    "OIDC_RP_SCOPES": "openid email profile groups offline_access",

    # Security (all enabled)
    "OIDC_USE_NONCE": True,
    "OIDC_USE_PKCE": True,
    "OIDC_STATE_SIZE": 32,
    "OIDC_NONCE_SIZE": 32,

    # User management
    "OIDC_CREATE_USER": False,  # Only existing users

    # Custom services
    "OIDC_SERVICE": "myapp.services.EnterpriseOIDCService",
    "OIDC_USER_SERVICE": "myapp.services.EnterpriseUserService",
    "OIDC_JWT_VERIFIER_SERVICE": "myapp.services.EnterpriseJWTVerifier",

    # Custom serializers
    "AUTHENTICATION_OUTPUT_SERIALIZER": "myapp.serializers.EnterpriseUserSerializer",
    "GET_USER_OUTPUT_SERIALIZER": "myapp.serializers.DetailedEnterpriseUserSerializer",

    # JWT claims validation
    "OIDC_ID_TOKEN_CLAIMS": ("iss", "exp", "iat", "sub", "email", "email_verified", "groups"),
    "OIDC_ACCESS_TOKEN_CLAIMS": ("iss", "aud", "exp", "uid", "scp"),

    # Extended cookie lifetime
    "OIDC_REFRESH_TOKEN_COOKIE_MAX_AGE": 60 * 60 * 24 * 14,  # 14 days
}

    Environment Variables¶

    Using django-environ¶

    # settings.py
import environ

env = environ.Env()
environ.Env.read_env()  # Read .env file

TINKUN_OIDC = {
    "OKTA_DOMAIN": env("OKTA_DOMAIN"),
    "OIDC_RP_CLIENT_ID": env("OIDC_RP_CLIENT_ID"),
    "OIDC_RP_CLIENT_SECRET": env("OIDC_RP_CLIENT_SECRET"),
    "OIDC_AUTHENTICATION_CALLBACK_URL": env("OIDC_AUTHENTICATION_CALLBACK_URL"),
}

    .env File¶

    # .env
OKTA_DOMAIN=dev-123456.okta.com
OIDC_RP_CLIENT_ID=0oa1234567890abcdef
OIDC_RP_CLIENT_SECRET=very-secret-string-from-okta
OIDC_AUTHENTICATION_CALLBACK_URL=https://yourapp.com/auth/callback

    Important: Never commit .env files to version control.

    Validation¶

    The package validates configuration on startup:

    • Required settings are present
    • Endpoint URLs are valid
    • Service paths are importable
    • Serializer paths are importable

    Invalid configuration will raise ImproperlyConfigured exception.

    For implementation details, see:

    • Authentication - OIDC authentication flow
    • Services - Service layer architecture
    • API - API endpoints and usage
